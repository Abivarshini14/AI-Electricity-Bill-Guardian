# API Documentation

Base URL (local): `http://localhost:8000`

Interactive, always-current API docs are auto-generated by FastAPI and available at:

- Swagger UI: `GET /docs`
- ReDoc: `GET /redoc`
- OpenAPI schema: `GET /openapi.json`

These reflect every route, request/response schema, and validation rule exactly, so this file gives a high-level map rather than duplicating field-level detail.

## Authentication

All endpoints except `/api/auth/register`, `/api/auth/login`, and `/health` require a Bearer JWT:

```
Authorization: Bearer <access_token>
```

Obtain a token from `POST /api/auth/register` or `POST /api/auth/login`.

| Prefix | Purpose |

| `/api/payments` | simulate payment, history, receipt PDF |
| `/api/budgets` | set/get budget |
| `/api/savings-goals` | create/list savings goals |
| `/api/dashboard/{property_id}/summary` | aggregated usage pace, health score, bill shock, budget status |
| `/api/notifications` | list, unread count, mark read |
| `/api/ai/chat` | Grok-powered assistant with deterministic fallback |
| `/api/challenges`, `/api/streaks`, `/api/away-mode`, `/api/outages` | extra smart-usage features |

| `/health` | health check, no auth required |

## Ownership & security rules

- Every property-scoped endpoint verifies the authenticated user (from the JWT) owns the property, or is an ADMIN. The frontend never sends a trusted `user_id`.
- Admin-only routes use a separate `get_current_admin` dependency and return `403` for non-admins.
- Passwords are hashed with bcrypt via Passlib; plaintext passwords are never stored or logged.
- File uploads (profile photos, complaint attachments) are validated by extension and size before being written to disk.

## Complaint status flow

`Submitted → Under Review → In Progress → Resolved → Closed`

Any status may transition directly to `Closed`. The backend rejects invalid transitions (e.g. `Submitted → Resolved`) with a `400` error.
